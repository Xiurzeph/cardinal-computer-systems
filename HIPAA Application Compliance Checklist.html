HIPAA Application Compliance Checklist

Developer Evaluation Form

Why this Checklist is Critical

This questionnaire is essential for identifying potential gaps in the application's design that could compromise Protected Health Information (ePHI). By verifying the presence of technical controls (like encryption, audit logging, and access control), you ensure the developer, as a Business Associate, meets the minimum necessary requirements of the HIPAA Security Rule. Using 'Planned' helps determine if any major compliance efforts are still outstanding.

Category Question Yes No Planned Notes/Comments
Access Control Does the application require a unique, unshared user ID for every individual accessing ePHI? (164.312(a)(2)(i))
Access Control Is an automatic logoff mechanism implemented after a predefined period of inactivity? (164.312(a)(2)(iii))
Access Control Does the application enforce granular, role-based access control (RBAC) to ensure users only access the minimum necessary ePHI? (164.306(a)(4))
Transmission Is all ePHI encrypted (e.g., using TLS 1.2 or higher) when it is transmitted over an electronic network? (164.312(e)(1))
Transmission Are mechanisms in place to ensure that ePHI has not been improperly altered or destroyed during electronic transmission? (164.312(c)(1))
Storage & Integrity Is all stored ePHI encrypted (e.g., database fields, file storage)? (164.312(a)(2)(iv))
Storage & Integrity Does the application environment have a documented and tested data backup and disaster recovery plan for ePHI? (164.308(a)(7)(i))
Audit Controls Does the application record activity in systems that contain ePHI (logins, access attempts, modifications) with comprehensive audit logs? (164.312(b))
Audit Controls Are audit logs retained and protected from modification for the required minimum period (typically six years)? (164.316(b)(1)(ii))
Physical Security Is access to the physical location where the application/database is hosted secured with appropriate access controls (e.g., locked doors, surveillance)? (164.310(a)(1))
Physical Security Are physical access logs reviewed and retained for all personnel entering the data center/hosting facility? (164.310(a)(2)(ii))
Environmental Mgt. Are environmental controls (HVAC, fire suppression, power redundancy, e.g., UPS/Generator) in place and regularly tested? (164.310(a)(2)(i))
Media Disposal Are formal policies and procedures in place for the secure disposal or reuse of electronic media (hard drives, backup tapes) that stored ePHI? (164.310(d)(2)(iv))
Administrative Has the developer established and documented formal security policies and procedures regarding ePHI handling? (164.308(a)(1)(i))
Administrative Has a comprehensive Risk Analysis been performed on the application/hosting environment? (164.308(a)(1)(ii)(A))
Administrative If the developer hosts the application or accesses ePHI, is a signed Business Associate Agreement (BAA) in place?
